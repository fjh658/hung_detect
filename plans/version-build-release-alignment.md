# Version / Build / Release Alignment for hung_detect

**Status: Completed (2026-02-17, updated 2026-02-27, code cleanup 2026-02-27)**

## Context

Recent iterations introduced SwiftPM tests, source layout changes, and new diagnosis flags.
This round focused on removing version drift and release friction:

1. `--version` was missing initially.
2. Version value was duplicated across Makefile / formula / runtime output.
3. Formula maintenance (`version`, artifact name, sha256) was manual and error-prone.
4. Build path had become inconsistent (manual fallback logic and old docs).

Goal: establish one clear version source and deterministic packaging flow.

### Update (2026-02-27): Build Time & Traceability

Added build-time stamp and scan/start time to all output modes for traceability ‚Äî when debugging long-running monitor sessions, knowing which version and when it was built is critical.

## Final Decisions

1. **Single source of truth for runtime/app version**
   Use `Sources/hung_detect/Version.swift`:
   ```swift
   let toolVersion = "0.4.0"
   ```

2. **Runtime `--version` comes from code, not Makefile**
   `hung_detect --version` / `-v` prints `toolVersion`.

3. **Makefile reads version from `Version.swift`**
   Makefile no longer owns a hard-coded `VERSION ?= ...`.
   It parses `toolVersion` and uses it for tarball naming.

4. **Formula file is generated from template during packaging**
   `Formula/hung-detect.rb.tmpl` contains placeholders:
   - `__VERSION__`
   - `__SHA256__`

   `make package` renders `Formula/hung-detect.rb` with:
   - version parsed from `Version.swift`
   - sha256 from freshly built tarball

5. **Keep build path simple**
   `make build` uses SwiftPM multi-arch only:
   `swift build --arch arm64 --arch x86_64`

### Output Traceability (2026-02-27)

All output modes now include version, build time, and scan/start time:

| Output mode | Format |
|---|---|
| `--version` | `hung_detect 0.5.1 (built <build_time>)` |
| Single-shot table | Header line: `hung_detect 0.5.1 (built <build_time>) scanned <scan_time>` |
| Single-shot JSON | Top-level `"version"`, `"build_time"`, `"scan_time"` fields |
| Monitor table | Start banner: `hung_detect 0.5.1 (built <build_time>) started <start_time>` |
| Monitor JSON | `monitor_start`/`monitor_stop` events include `"version"`, `"build_time"`, `"timestamp"` |

### Code Quality Cleanup (2026-02-27)

Comprehensive code review followed by targeted refactoring:

1. **Shared DateFormatter instances** ‚Äî Extracted 3 global formatter singletons (`isoFormatter`, `timeOnlyFormatter`, `fileTimestampFormatter`) to replace repeated per-call creation. All callers confirmed main-thread-only.
2. **Deduplicated diagnosis dispatch** ‚Äî `runSingleShot` and `triggerAsync` shared ~80 lines of identical dispatch logic. Extracted `dispatchDiagnosis()` + `DiagResultCollector` class for thread-safe result aggregation.
3. **Merged spindump methods** ‚Äî `runSpindumpPid` and `runSpindumpSystem` (~60 lines each, nearly identical) merged into single `runSpindump(pid:name:tool:targetArgs:duration:intervalMs:extraTimeout:outdir:timestamp:)`.
4. **Removed unnecessary lock on `C.enabled`** ‚Äî Color flag is set once in `main()` then read-only, all access on main thread. NSLock (`stateLock`) removed, simplified to plain `static var`.
5. **Fixed `renderProcessTable` indentation** ‚Äî Method body was missing one level of indentation relative to its class declaration.

Concurrency audit confirmed no deadlocks or variable synchronization issues:
- 5 independent NSLock instances, no nested acquisition anywhere.
- MonitorEngine state protected by `dispatchPrecondition(.onQueue(.main))`.
- DispatchGroup enter/leave strictly paired in all code paths.
- DateFormatters accessed exclusively from main thread.

Issues reviewed and intentionally left as-is:
- Single-shot vs monitor scan logic overlap (~15 lines) ‚Äî different output types (`ProcEntry` vs `ProcessSnapshot`) make extraction more complex than the duplication.
- Hand-rolled JSON ‚Äî all user-controlled strings pass through `escJSON`, no missing escapes found; switching to Codable would be high-churn for low benefit at current scale.

## Operational Workflow (Now)

For a new release:

1. Update `Sources/hung_detect/Version.swift`
2. Run `make package` (or `make package INCLUDE_DSYM=1` when release artifact should include `.dSYM`)
3. Commit:
   - source changes
   - `dist/hung-detect-<version>-macos-universal.tar.gz`
   - generated `Formula/hung-detect.rb`
4. Push/tag/release

This keeps runtime version, tarball name, and formula metadata synchronized.
